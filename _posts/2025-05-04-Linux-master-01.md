## 명령어
- XFS 파일 시스템을 생성: mkfs.xfs
- 셸의 정보를 확인: chsh -l
- yum을 통해 설치된 telnet-server 패키지를 제거: yum remove telnet-server
- 프로세스 ID(PID)로 우선순위를 변경: renice {nice값} -p {PID}
- 프로세스명 우선순위를 변경: nice -n {nice값} {명령어}
- 실시간 CPU 사용량 확인: top
- SSH 접속 시 사용할 포트 번호 지정: ssh -p
- 디스크 장치에 할당된 UUID를 확인: blkid
- 게이트웨이 주소 설정: route add default gw {게이트웨이 주소} {인터페이스}
  * (신형, 권장 방식: ip route add default via {게이트웨이 주소} dev {인터페이스})
- 모든 사용자/그룹의 쿼터 현황 확인: repquota
- 사용자의 쿼터 사용량 확인: quota
- 사용자의 쿼터 설저 편집: edquota
- 네트워크 인터페이스 카드의 상세 설정과 상태를 조회, 변경: ethtool
- 네트워크 연결 상태, 포트, 라우팅 테이블, 인터페이스 통계 등 출력: netstat
  * netstat 대체 명령어: ss (socket statistics)
- 실시간으로 CPU, 메모리 사용량, 실행중인 프로세스 상태를 표시: top
- 주소, 라우팅, 링크 상태, 인터페이스 제어: ip
- ip 설정과 네트워크 인터페이스(ip, mac, broadcast) 상태 확인: ifconfig

## 권한 설정
### 파일/디렉터리의 권한과 소유권을 다루는 명령어
- ls: 파일/디렉터리 목록 확인
- chmod: 권한 변경
- chgrp: 그룹 변경
- chown: 소유자/그룹 변경

### 숫자 권한 설정법
- 사용자 유형: u(파일 소유자) / g(파일 소유 그룹) / o(그외 사용자)
- 권한 종류: r(4, 읽기) / w(2, 쓰기) / x(1, 실행)
- 예시: `chmod 754 file.txt`
  - 소유자: 7 = 읽기 + 쓰기 + 실행
  - 그룹: 5 = 읽기 + 실행
  - 그외: 4 = 읽기
 
### 기호 권한 설정법
- 대상: u (소유자), g (그룹), o (기타), a (모두)
- 연산자: + (추가), - (제거), = (지정)
- 권한: r (읽기), w (쓰기), x (실행)
- 예시: `chmod a=r program.c`
  - 모든 사용자에게 읽기만 설정, 기존 권한은 초기화
 
### 옵션

|옵션|설명|예시|
|:----:|:----:|:----:|
| **`-R`** | 하위 디렉터리까지 **재귀적으로** 변경 | `chown -R user:group /data` |
| **`-f`** | 에러 메시지를 **출력하지 않음** | `chown -f user file` |
| **`-v`** | 변경된 내용을 **자세히 출력** | `chown -v user file` |
| **`-r`** | Deprecated, 더 이상 사용되지 않음 (과거의 Recursive) | |
| **`-c`** | Changes, 변경된 항목만 출력 | |
| **`--from=current_owner`** | **현재 소유자**가 특정 사용자일 때만 변경  | `chown --from=root user file` |
| **`--reference=file`** | 다른 파일의 소유자/그룹을 **참조하여** 변경 | `chown --reference=template file` |


## 환경변수
- USER: 현재 로그인한 사용자 이름
- HOME: 사용자의 홈 디렉토리 경로
- SHELL: 기본으로 사용하는 셸 프로그램 (사용자의 로그인 셸이 기록)
- PWD: 현재 작업 중인 디렉토리
- PS1: 기본 프롬프트 (ex: $, #)
- PS2: 줄이 계속될 때 나오는 2차 프롬프트 (역슬래시 사용)
- PS3: select 명령에서 선택지 앞에서 나오는 프롬프트
- PS4: set -x 디버깅 시 앞에서 붙는 표시

## 라이브러리
- Xlib: X Window System을 위한 기본 C 인터페이스 라이브러리
- XCB: Xlib보다 빠르고 모듏화된 X 프로토콜 통신 라이브러리
- FLTK: C++로 작성된 가볍고 빠른 GUI 툴킷 
- GTX+: GNOME 데스크탑에서 주로 사용
- QT: KDE 데스크탑 환경에서 주로 사용
- Wayland: X를 대체, 차세대 디스플레이 서버 프로토콜
- Cairo: 2D 벡터 그래픽 랜더링용 라이브러리
- Pango: 텍스트 레이아웃 및 랜더링
- Mesa: 오픈소스 OpenGL 구현체 (그래픽 하드웨어 가속)


## OSI 7계층 

| OSI 계층 | 계층 이름 (한글) | 대표 기능 | 대표 장비 예시 | 프로토콜 예시 |
|:---:|:---:|:---:|:---:|:---:|
| 7 | 응용 계층 (Application)   | 사용자 인터페이스, 웹 서비스 | **L7 스위치**, 애플리케이션 게이트웨이 | TELNET, FTP, SMTP, HTTP |
| 6 | 표현 계층 (Presentation)  | 암호화, 압축, 변환 | (일반 장비 없음, 소프트웨어 처리) | |
| 5 | 세션 계층 (Session)       | 세션 설정 및 관리 | (일반 장비 없음, 소프트웨어 처리) | |
| 4 | 전송 계층 (Transport)     | TCP/UDP 포트 제어 | 고급 방화벽 (포트 기반 필터링) | TCP, UCP |
| 3 | 네트워크 계층 (Network)     | IP 주소 기반 라우팅 | **라우터**, L3 스위치, 방화벽 | IP, ARP, ICMP |
| 2 | 데이터 링크 계층 (Data Link) | MAC 주소 기반 스위칭 | **스위치(L2)**, 브리지 | HDLC, X.25, Ethernet |
| 1 | 물리 계층 (Physical)      | 전기 신호, 케이블 전송 | **허브**, 리피터, 케이블 | X.21, RS-232 | 

- MAC 주소 변환은 ARP

## 주소 계산
- 네트워크 주소: IP 주소 OR (NOT 서브넷 마스크)
- 브로드캐스트 주소: IP 주소 AND 서브넷 마스크
- 게이트웨이 주소: 네트워크 주소 ~ 브로드캐스트 주소 사이 유효 주소 (보통 첫번째 유효 주소)
  
  | 항목 | 값 |
  |:---:|:---:|
  | **IP 주소** | 192.168.5.66 |
  | **서브넷 마스크** | 255.255.255.192 (/26) |
  | **네트워크 주소** | 192.168.5.64 |
  | **브로드캐스트 주소** | 192.168.5.127 |
  | **게이트웨이 주소 (보통)** | 192.168.5.65 |

### LAN 구성 방식

|구성방식|특징|장점|단점|
|:---:|:---:|:---:|:---:|
| **버스형 (Bus)** | 단일 케이블에 모든 노드 연결 | 설치 비용이 저렴, 구조가 단순 | 케이블 단선 시 전체 네트워크 장애, 충돌 발생 가능 |
| **성형 (Star)** | 중앙 허브 또는 스위치에 모든 노드 연결 | 장애 노드 영향 최소화, 성능 우수 | 중앙 허브 장애 시 전체 네트워크 마비, 설치 비용 높음 |
| **링형 (Ring)** | 노드가 **순환 형태**로 연결 | 충돌 없이 데이터 전송 | 노드 추가/제거 어려움, 단선 시 전체 네트워크 장애 |
| **메시형 (Mesh)** | 모든 노드가 상호 연결 | 높은 신뢰성, 장애 복구 용이 | 설치 복잡, 비용 높음, 장애 발생 시 고장 지점 파악 어려움 |
| **트리형 (Tree)** | 성형 + 버스형 혼합, 계층적 구조 | 대규모 네트워크에 적합 | 트렁크 케이블 문제 시 전체 장애 |
| **혼합형 (Hybrid)** | 둘 이상의 토폴로지 혼합 | 유연한 구조 | 복잡한 구성, 관리 어려움 |
